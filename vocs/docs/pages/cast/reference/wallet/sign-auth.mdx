## cast wallet sign-auth

EIP-7702 sign authorization

```bash
$ cast wallet sign-auth --help
```

```txt
Usage: cast wallet sign-auth [OPTIONS] &lt;ADDRESS&gt;

Arguments:
  &lt;ADDRESS&gt;
          Address to sign authorization for

Options:
  -r, --rpc-url &lt;URL&gt;
          The RPC endpoint, default value is http://localhost:8545
          
          [env: ETH_RPC_URL=]

  -k, --insecure
          Allow insecure RPC connections (accept invalid HTTPS certificates).
          
          When the provider's inner runtime transport variant is HTTP, this
          configures the reqwest client to accept invalid certificates.

      --flashbots
          Use the Flashbots RPC URL with fast mode
          (&lt;https://rpc.flashbots.net/fast&gt;).
          
          This shares the transaction privately with all registered builders.
          
          See:
          &lt;https://docs.flashbots.net/flashbots-protect/quick-start#faster-transactions&gt;

      --jwt-secret &lt;JWT_SECRET&gt;
          JWT Secret for the RPC endpoint.
          
          The JWT secret will be used to create a JWT for a RPC. For example,
          the following can be used to simulate a CL `engine_forkchoiceUpdated`
          call:
          
          cast rpc --jwt-secret &lt;JWT_SECRET&gt; engine_forkchoiceUpdatedV2
          '["0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc",
          "0x6bb38c26db65749ab6e472080a3d20a2f35776494e72016d1e339593f21c59bc"]'
          
          [env: ETH_RPC_JWT_SECRET=]

      --rpc-timeout &lt;RPC_TIMEOUT&gt;
          Timeout for the RPC request in seconds.
          
          The specified timeout will be used to override the default timeout for
          RPC requests.
          
          Default value: 45
          
          [env: ETH_RPC_TIMEOUT=]

      --rpc-headers &lt;RPC_HEADERS&gt;
          Specify custom headers for RPC requests
          
          [env: ETH_RPC_HEADERS=]

      --nonce &lt;NONCE&gt;
          

      --chain &lt;CHAIN&gt;
          

  -h, --help
          Print help (see a summary with '-h')

  -j, --threads &lt;THREADS&gt;
          Number of threads to use. Specifying 0 defaults to the number of
          logical cores
          
          [aliases: --jobs]

Wallet options - raw:
  -f, --from &lt;ADDRESS&gt;
          The sender account
          
          [env: ETH_FROM=]

  -i, --interactive
          Open an interactive prompt to enter your private key

      --private-key &lt;RAW_PRIVATE_KEY&gt;
          Use the provided private key

      --mnemonic &lt;MNEMONIC&gt;
          Use the mnemonic phrase of mnemonic file at the specified path

      --mnemonic-passphrase &lt;PASSPHRASE&gt;
          Use a BIP39 passphrase for the mnemonic

      --mnemonic-derivation-path &lt;PATH&gt;
          The wallet derivation path.
          
          Works with both --mnemonic-path and hardware wallets.

      --mnemonic-index &lt;INDEX&gt;
          Use the private key from the given mnemonic index.
          
          Used with --mnemonic-path.
          
          [default: 0]

Wallet options - keystore:
      --keystore &lt;PATH&gt;
          Use the keystore in the given folder or file
          
          [env: ETH_KEYSTORE=]

      --account &lt;ACCOUNT_NAME&gt;
          Use a keystore from the default keystores folder
          (~/.foundry/keystores) by its filename
          
          [env: ETH_KEYSTORE_ACCOUNT=]

      --password &lt;PASSWORD&gt;
          The keystore password.
          
          Used with --keystore.

      --password-file &lt;PASSWORD_FILE&gt;
          The keystore password file path.
          
          Used with --keystore.
          
          [env: ETH_PASSWORD=]

Wallet options - hardware wallet:
  -l, --ledger
          Use a Ledger hardware wallet

  -t, --trezor
          Use a Trezor hardware wallet

Wallet options - remote:
      --aws
          Use AWS Key Management Service

      --gcp
          Use Google Cloud Key Management Service

Display options:
      --color &lt;COLOR&gt;
          The color of the log messages

          Possible values:
          - auto:   Intelligently guess whether to use color output (default)
          - always: Force color output
          - never:  Force disable color output

      --json
          Format log messages as JSON

  -q, --quiet
          Do not print log messages

  -v, --verbosity...
          Verbosity level of the log messages.
          
          Pass multiple times to increase the verbosity (e.g. -v, -vv, -vvv).
          
          Depending on the context the verbosity levels have different meanings.
          
          For example, the verbosity levels of the EVM are:
          - 2 (-vv): Print logs for all tests.
          - 3 (-vvv): Print execution traces for failing tests.
          - 4 (-vvvv): Print execution traces for all tests, and setup traces
          for failing tests.
          - 5 (-vvvvv): Print execution and setup traces for all tests,
          including storage changes.
```
