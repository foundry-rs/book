# Quick Actions

---

## Forge

[`forge`](/forge/overview) helps you build, test, debug, deploy and verify smart contracts on ZKsync.

:::code-group

```bash [Initialize ZKsync project]
# Initializes a project called `Counter`
forge init
```

```bash [Run ZKsync tests]
# Run tests on zkEVM using --zksync flag
forge test --zksync
# You can also test against ZKsync mainnet/testnet
forge test --fork-url https://mainnet.era.zksync.io --zksync
```

```bash [Deploy to ZKsync]
# Deploy contracts to ZKsync network using --zksync flag
# Set the private key in the env
export PRIVATE_KEY="0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80"
# Run the script and broadcast to ZKsync testnet
forge script script/Counter.s.sol --rpc-url https://testnet.era.zksync.dev --broadcast --private-key $PRIVATE_KEY --zksync
```

```bash [Dual compilation]
# Compile for both EVM and zkEVM
forge build --zksync
# View compilation artifacts for both
ls out/ 
ls zkout/
```

:::

## Cast

[`cast`](/cast/overview) works with ZKsync networks to interact with contracts and query blockchain state.

:::code-group

```bash [Call ZKsync contract]
# Call a contract function on ZKsync mainnet
cast call 0x32400084C286CF3E17e7B677ea9583e60a000324 \
"balanceOf(address)" 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 \
--rpc-url https://mainnet.era.zksync.io
```

```bash [Send ZKsync transaction]
# Send a transaction to ZKsync testnet
export PRIVATE_KEY="0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80"
cast send 0x70997970C51812dc3A010C7d01b50e0d17dc79C8 \
--value 10000000 \
--private-key $PRIVATE_KEY \
--rpc-url https://testnet.era.zksync.dev
```

:::
